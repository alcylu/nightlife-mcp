<feature>
  <name>Search Query Normalization Utility</name>
  <files>src/utils/normalize.ts, src/utils/normalize.test.ts</files>
  <behavior>
    normalizeQuery(raw: string): string
    - Strips accents via NFD decomposition + combining diacritics removal
    - Collapses all whitespace (spaces, tabs, etc.)
    - Lowercases the result
    - Preserves digits and basic ASCII

    Test cases (from requirements + success criteria):
    - NORM-01: normalizeQuery('e') -> 'e' (accent stripped)
    - NORM-01: normalizeQuery('o') -> 'o' (macron stripped)
    - NORM-01: normalizeQuery('u') -> 'u' (umlaut stripped)
    - NORM-02: normalizeQuery('CE LA VI') -> 'celavi' (accent + space collapsed)
    - NORM-02: normalizeQuery('1 OAK') -> '1oak' (space collapsed)
    - NORM-03: normalizeQuery('1oak') -> '1oak' (digits preserved)
    - NORM-04: normalizeQuery('CeLaVi') -> 'celavi' (case normalized)
    - Edge: normalizeQuery('') -> '' (empty string)
    - Edge: normalizeQuery('  ') -> '' (whitespace only)

    stripAccents(s: string): string
    - Strips accents but preserves spaces and original casing
    - Test: stripAccents('CE LA VI') -> 'CE LA VI'
    - Test: stripAccents('Shinjuku') -> 'Shinjuku' (macron stripped, space preserved)
  </behavior>
  <implementation>
    Use `String.prototype.normalize('NFD')` to decompose accented characters into base + combining diacritics, then strip the combining diacritical marks range (U+0300 to U+036F) with regex. This is the canonical Unicode solution — no lookup tables, no npm packages.

    normalizeQuery: NFD normalize -> strip diacritics -> collapse whitespace -> lowercase
    stripAccents: NFD normalize -> strip diacritics (preserves spaces and case)

    Follow the exact pattern from the research document's Code Examples section.
  </implementation>
</feature>
