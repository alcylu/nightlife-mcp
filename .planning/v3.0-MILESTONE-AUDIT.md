## Tech Debt Summary

### NORM-03 Scope Narrowing (Scoping Decision)

REQUIREMENTS.md defines NORM-03 as "Number-word equivalence — '1oak' matches 'oneoak'." The plan deliberately narrowed this to "digits preserved" (`normalizeQuery('1oak')` → `'1oak'`). No digit-to-word mapping was implemented. RESEARCH.md explicitly flagged this as potentially over-specified and recommended deferring.

**Options:**
1. Accept narrowed interpretation — mark NORM-03 fully satisfied (the fuzzy RPC provides a partial safety net)
2. Add digit-to-word mapping — ~5-line change to `normalize.ts` with a test case

### Venues Pass-1 Needle Inconsistency

Venues uses `sanitizeIlike` for pass-1 client-side needle while events/performers use `normalizeQuery`. Functionally correct (RPC catches misses) but inconsistent pattern.

### Nyquist Validation Gaps
